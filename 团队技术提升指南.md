## 一、布局架构原则（本次问题的核心教训）

### 1.1 单一职责原则 — 一个容器只做一件事

**错误示范（修复前）:**
```html
<!-- baseof.html -->
<div class="main-wrapper">
  <div class="container">
    {{ block "main" . }}{{ end }}
  </div>
</div>

<!-- home.html -->
<div class="main-wrapper" id="content">
  <div class="container">
    <div class="content-grid">
      <!-- 内容 -->
    </div>
  </div>
</div>
```

**结果**: 双重嵌套导致：
- `max-width` 被应用两次
- `padding` 叠加（1rem + 1.5rem = 2.5rem）
- 时间线 `padding-left: 2rem` 计算基准错误

**正确做法:**
```html
<!-- baseof.html —— 只负责 block 占位 -->
{{ block "main" . }}{{ end }}

<!-- home.html —— 负责具体布局 -->
<div class="main-wrapper" id="content">
  <div class="container">
    <div class="content-grid">
      <!-- 内容 -->
    </div>
  </div>
</div>
```

### 1.2 布局层级约定

```
body
  └── site-header      (sticky, z-index: 100)
  └── hero-section     (100vh, 可选)
  └── main-wrapper     (max-width, margin: auto)
      └── container    (padding, width: 100%)
          └── content-grid  (grid/flex 布局)
              └── main / aside
  └── site-footer
```

**规则**: 每一层只负责一个维度（宽度 / 间距 / 布局方式）。

## 二、CSS 定位最佳实践

### 2.1 绝对定位的坐标系陷阱

**问题**: 时间线圆点使用 `position: absolute; left: -2rem`，但父容器有 `padding-left: 2rem`。

**分析**:
```
.timeline-container {
  padding-left: 2rem;    /* 内容从 2rem 开始 */
  position: relative;    /* 定位上下文 */
}

.timeline-line {
  position: absolute;
  left: 0;               /* 贴左边 = 容器真实左边缘 */
}

.timeline-dot {
  position: absolute;
  left: -2rem;           /* 相对于 padding-left: 2rem 的内容区 */
  /* 实际位置 = 2rem(padding) - 2rem(left) = 0 */
  /* 但圆点宽度 10px，transform: translateX(-4px) 让它偏左 */
}
```

**修复方案**:
```scss
.timeline-container {
  padding-left: 2rem;     // 给竖线留空间
  position: relative;
}

.timeline-line {
  position: absolute;
  left: 0;                // 贴容器左边缘
  top: 0.5rem;            // 从第一个圆点中心开始
}

.timeline-dot {
  position: absolute;
  left: -2rem;            // 回到容器左边缘
  transform: translateX(-5px);  // 向左偏移半个圆点宽度，精确居中
}
```

### 2.2 使用 transform 而非 margin 做微调

| 属性 | 是否触发重排 | 是否影响兄弟元素 | 推荐度 |
|------|-----------|--------------|-------|
| `margin` | 是 | 是 | ⭐⭐ |
| `transform` | 否 | 否 | ⭐⭐⭐⭐⭐ |
| `position + left/top` | 是 | 否 | ⭐⭐⭐ |

**原则**: 做视觉微调时优先使用 `transform`，性能更好且不破坏布局流。

## 三、Hugo 模板开发规范

### 3.1 变量作用域管理

**错误**:
```go
{{ range $date, $pages := $paginator.Pages.GroupByDate "2006.01.02" }}
  {{ range $index, $page := $pages }}
    <!-- $index 是内层循环的索引，不是组的索引！ -->
    <div class="{{ if eq $index 0 }}featured{{ end }}">
  {{ end }}
{{ end }}
```

**正确**:
```go
{{ $grouped := $paginator.Pages.GroupByDate "2006.01.02" }}
{{ range $i, $group := $grouped }}
  <!-- $i 是组的索引，$group.Key 是日期 -->
  <div class="timeline-group{{ if eq $i 0 }} featured{{ end }}">
    <div class="timeline-date">{{ $group.Key }}</div>
    {{ range $group.Pages }}
      <!-- 文章循环 -->
    {{ end }}
  </div>
{{ end }}
```

### 3.2 避免在模板中硬编码逻辑

**错误**:
```go
<!-- 直接在 range 中判断，难以维护 -->
{{ if and (eq $paginator.PageNumber 1) (eq $index 0) }}
```

**正确**:
```go
<!-- 提取为变量，语义清晰 -->
{{ $isFirstPage := eq $paginator.PageNumber 1 }}
{{ $isFirstGroup := eq $i 0 }}
<div class="timeline-group{{ if and $isFirstPage $isFirstGroup }} featured{{ end }}">
```

## 四、SCSS 架构规范

### 4.1 文件组织

```
assets/css/
├── main.scss              # 入口文件，只负责 @import
└── partials/
    ├── _variables.scss    # CSS 变量、SCSS 变量、mixins
    ├── _base.scss         # Reset、基础样式
    ├── _layout.scss       # 容器、网格系统
    ├── _header.scss       # 导航
    ├── _hero.scss         # Hero 区域
    ├── _timeline.scss     # 时间线组件
    ├── _components.scss   # 通用组件（按钮、卡片等）
    └── _widgets.scss      # 侧边栏小部件
```

### 4.2 命名规范

| 类型 | 前缀 | 示例 |
|------|------|------|
| 布局 | 无 | `.main-wrapper`, `.content-grid` |
| 组件 | 无 | `.timeline-card`, `.search-widget` |
| 状态 | 无 | `.featured`, `.active`, `.hidden` |
| 工具类 | 无 | `.span-col-2` |
| JS 钩子 | `js-` | `.js-menu-toggle` |

### 4.3 z-index 管理

**禁止随意写数字！** 使用变量或层级约定：

```scss
// _variables.scss
$z-base: 1;
$z-content: 2;
$z-overlay: 10;
$z-header: 100;
$z-dropdown: 200;
$z-modal: 1000;
$z-toast: 2000;
```

## 五、响应式设计规范

### 5.1 Mobile-First 原则

```scss
// 先写移动端样式
.timeline-container {
  padding-left: 2rem;
}

// 再用 media query 覆盖桌面端
@media (min-width: 768px) {
  .timeline-container {
    padding-left: 2.5rem;
  }
}
```

**为什么**: 移动端用户更多，且移动端的约束更强（屏幕小）。先解决移动端，再渐进增强到桌面端。

### 5.2 断点统一

```scss
// _variables.scss
$breakpoint-sm: 640px;
$breakpoint-md: 768px;
$breakpoint-lg: 1024px;
$breakpoint-xl: 1280px;

@mixin md { @media (min-width: $breakpoint-md) { @content; } }
@mixin lg { @media (min-width: $breakpoint-lg) { @content; } }
```

## 六、性能优化清单

### 6.1 CSS
- [ ] 使用 `transform` 和 `opacity` 做动画（GPU 加速）
- [ ] 避免在动画中改变 `width`、`height`、`top`、`left`
- [ ] 使用 `will-change` 谨慎（只在需要时添加，动画结束后移除）
- [ ] 压缩 CSS（production 环境）

### 6.2 图片
- [ ] 使用 WebP/AVIF 格式
- [ ] 提供多尺寸 srcset
- [ ] 懒加载非首屏图片

### 6.3 JavaScript
- [ ] 使用 `type="module"` 和代码分割
- [ ] 第三方库使用 CDN + integrity
- [ ] 避免阻塞渲染的脚本

## 八、本次修复的关键教训

| # | 问题 | 根因 | 预防措施 |
|---|------|------|---------|
| 1 | 布局错乱 | 双重 `.main-wrapper` 嵌套 | 审查 baseof.html 和具体页面的结构关系 |
