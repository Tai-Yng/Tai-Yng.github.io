## 修复 CSS 语法错误 (2026-05-14 19:43)

### 问题描述
文章页面（post/single.html）样式显示异常，只有首页正常。

### 根本原因
`main.css` 文件中 `[data-theme="dark"]` 样式块缺少开头的 `[` 符号：
```css
/* 错误 */
}}--bg-body: #09090b;...

/* 正确 */
}}[data-theme="dark"]{--bg-body: #09090b;...
```

### 修复方法
使用 sed 命令修复两个位置的 CSS 文件：
- `themes/kopi/assets/css/main.css`
- `public/css/main.css`

```bash
sed -i 's/}}--bg-body:/}}[data-theme="dark"]{--bg-body:/g' main.css
```

## 修复首页中间空白区域 (2026-05-14 19:45)

### 问题描述
首页中间有一大块空白区域（红框标注），不是设计上的留白，而是布局错误。

### 根本原因
`.blog-feed` 在桌面端设置了 `grid-template-columns: 1fr 1fr`（两列布局），但时间线内容只有一列，导致右侧列空白。

### 修复方法
将 `.blog-feed` 的桌面端布局从两列改回单列：
- `themes/kopi/assets/css/partials/_layout.scss`
- `public/css/main.css`

```css
/* 修复前 */
.blog-feed { grid-template-columns: 1fr 1fr; }

/* 修复后 */
.blog-feed { grid-template-columns: 1fr; }
```

## 修复 Posts 列表页卡片过长 (2026-05-14 20:02)

### 问题描述
Posts 列表页（`/posts/`）的卡片显示完整文章内容，导致卡片非常长、页面冗余。

### 根本原因
`post-card.html` 模板使用了 `$page.Content`（完整内容）而不是 `$page.Summary`（摘要），且截断长度长达 220-300 字符。

### 修复方法
1. 将 `$page.Content` 改为 `$page.Summary`
2. 缩短摘要长度：
   - 长标题：80 字符
   - 中等标题：100 字符
   - 短标题：120 字符

### 文件修改
- `layouts/_partials/post-card.html` - 使用 Summary 并缩短长度
